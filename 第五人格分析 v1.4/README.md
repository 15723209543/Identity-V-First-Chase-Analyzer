# Identity V First Chase Analyzer

> 第五人格监管者首抓分析器

## 项目简介

本项目是一个基于 **Visual Studio C++ / EasyX** 的《第五人格》监管者首追分析工具。程序会根据当前地图、求生者出生区域、求生者角色、监管者出生区域和监管者角色，计算监管者开局更可能首追哪一名求生者，并给出对应的首追概率、计算原因和行动建议。

程序采用图形化点击输入方式，启动后会打开 EasyX 窗口。用户通过鼠标选择地图、求生者位置、求生者角色、监管者位置和监管者角色，程序随后在右侧窗口显示完整分析结果，并将结果自动保存到 `result/` 文件夹。

本项目主要用于 C++ 面向对象程序设计、EasyX 图形界面练习、数据建模与简单策略分析展示，不代表游戏官方结论，也不保证完全符合任何版本的真实对局环境。

## 功能说明

本项目主要实现以下功能：

1. **地图选择**
   - 支持多个排位常见地图。
   - 小图采用 9 个区域划分。
   - 大图采用 12 个区域划分。
   - 左侧地图为简化区域图，用于交互选点和距离计算。

2. **求生者信息输入**
   - 支持依次选择 4 名求生者的出生区域。
   - 支持依次选择 4 名求生者角色。
   - 地图上会标记 `S1`、`S2`、`S3`、`S4`，并显示对应求生者名字。

3. **监管者信息输入**
   - 支持选择监管者出生 / 当前所在区域。
   - 支持选择监管者角色。
   - 地图上会标记 `H` 表示监管者位置。

4. **首追概率计算**
   - 根据求生者自身价值、监管者追击风格、地图区域压力、双方距离和角色克制关系进行综合评分。
   - 使用 softmax 将 4 名求生者的原始分转换为概率百分比。
   - 输出监管者对每名求生者的首追意愿。

5. **结果展示**
   - EasyX 右侧区域显示完整分析文本。
   - 输出每名求生者的角色指标、当前位置、首追概率和行动建议。
   - 输出监管者能力分析和对每名求生者的计算原因。

6. **结果保存**
   - 每次运行结束后自动生成分析结果文本。
   - 结果保存到 `result/` 文件夹。
   - 文件名按当前时间生成，不会覆盖上一次结果。

## 使用方法

程序所有输入均通过鼠标点击完成，不需要在终端输入。

输入顺序如下：

1. 点击本局地图。
2. 依次点击 4 名求生者所在区域。
3. 每选择一名求生者区域后，再点击该求生者角色编号。
4. 点击监管者所在区域。
5. 点击监管者角色编号。
6. 查看右侧完整分析结果。
7. 在 `result/` 文件夹中查看自动保存的结果文本。

结果文件命名格式示例：

```text
result/2026-06-14_14-50-00.txt
```

每次输出都会新建一个按当前时间命名的文件，不会覆盖历史结果。

## 算法说明

本项目对每名求生者计算 5 项分值：

1. **角色指标分**
   - 破译越高、牵制越低、首抓价值越高，越容易成为监管者首追目标。

2. **监管者风格分**
   - 追击型监管者更愿意抓自保弱、牵制差的角色。
   - 控场型监管者更重视高破译角色。
   - 守椅型监管者可能更重视救援位和能带节奏的角色。

3. **地图区域分**
   - 开阔区、低牵制区、电机价值高的区域会提高首追意愿。
   - 牵制资源多、转点舒适的区域会降低首追收益。

4. **距离分**
   - 监管者距离求生者越远，开局首追意愿越低。
   - 距离越近，首追可能性越高。

5. **克制分**
   - 根据监管者和求生者标签判断机制克制关系。
   - 例如反位移、反治疗、反道具、反破译等克制关系会提高首追权重。

核心原始分公式如下：

```cpp
raw_score = 50
    + character_score * 1.00
    + hunter_score * 0.80
    + map_score * 0.55
    + distance_score * 0.70
    + counter_score * 1.35;
```

随后使用 softmax 将四名求生者的原始分转换成概率百分比：

```text
probability_i = exp(score_i) / sum(exp(score_j))
```

最终概率越高，表示监管者越可能选择该求生者作为首追目标。

## 命名风格说明

本项目命名风格主要遵循以下规则：

1. **类名和结构体名使用小写英文与下划线**
   - 例如：`first_chase_analyzer`、`score_breakdown`

2. **函数名尽量体现动作或返回内容**
   - `find_survivor_data()` 表示查找求生者数据
   - `draw_result()` 表示绘制结果
   - `build_output_text()` 表示生成输出文本
   - `distance_penalty()` 表示计算距离惩罚

3. **成员变量使用下划线结尾**
   - 例如：`data_`、`region_id_`、`click_areas_`

4. **评分变量直接使用含义命名**
   - 例如：`character_score`、`counter_score`、`raw_score`

5. **静态数据常量使用 `k_` 前缀**
   - 例如：`k_survivor_data`、`k_hunter_data`、`k_map_data`

## 设计特点

### 1. 主函数较简洁

`main.cpp` 只负责设置编码并启动 `first_chase_app`，避免所有逻辑堆在主函数中。

### 2. 输入输出和算法分离

`easyx_ui` 只负责界面，`first_chase_analyzer` 只负责算法，`report_writer` 只负责结果文本，便于后续修改和维护。

### 3. 静态数据集中管理

求生者、监管者和地图数据分别放在独立头文件中，便于后续更新角色或地图。

### 4. 支持图形化点击输入

用户不需要在控制台输入编号，直接通过 EasyX 界面点击区域和角色即可完成输入。

### 5. 输出结果可保存

每次分析结果都会保存为独立文本文件，便于复盘和对比。

## 当前不足与未完成内容

当前版本核心功能已经完成，但仍有一些没有完成或可以继续完善的地方：

1. **没有自动读取 Excel 数据**
   - `data/identityv_scores.xlsx` 目前主要作为整理表。
   - 程序实际使用的数据仍写在 `include/survivor_data.h`、`include/hunter_data.h`、`include/map_data.h` 中。
   - 后续可以增加从 Excel、CSV 或 JSON 自动读取数据的功能。

2. **权重暂未提供图形化修改入口**
   - 当前评分权重写在 `src/first_chase_analyzer.cpp` 中。
   - 后续可以增加“权重设置界面”，允许用户手动调整模型参数。

3. **地图为简化区域图**
   - 左侧地图主要用于区域选择和距离计算，不追求完全还原游戏内地图比例。
   - 后续可以继续优化地图区域划分、位置坐标和视觉效果。

4. **暂未支持对局录像或截图自动识别**
   - 当前需要用户手动选择角色和位置。
   - 后续可以扩展为读取截图、录像或 OCR 识别，但当前版本未实现。

5. **结果输出格式仍可扩展**
   - 当前主要输出 `.txt` 文本文件。
   - 后续可以增加 Markdown、HTML、CSV 或图片报告导出。

6. **跨平台能力有限**
   - 当前依赖 Windows 和 EasyX。
   - Linux、macOS 暂不支持直接运行。

7. **数据版本需要持续维护**
   - 《第五人格》角色、地图和版本环境会更新。
   - 本项目评分数据需要根据实际版本继续调整。

## 预留开发接口说明

本项目已经预留了较清晰的模块结构，后续可以基于以下位置继续开发：

| 预留位置 | 可扩展方向 |

| `data_repository` | 改造成从 CSV、JSON、Excel 或数据库读取角色与地图数据 |

| `easyx_ui` | 增加设置界面、结果导出按钮、重新分析按钮 |

| `hunter_data.h` | 增加新监管者、调整监管者标签和评分 |
| `map_data.h` | 增加新地图、优化地图区域数据 |
| `first_chase_app` | 增加完整流程管理，例如重新开始、返回上一步、保存配置 |

建议后续优先补充“外部数据读取”和“权重可配置”功能，使项目不需要每次修改源码就能调整角色数据和模型参数。
