# Identity V First Chase Analyzer

> 第五人格监管者首抓分析器

## 项目简介

本项目是一个基于 **C++ / EasyX / Visual Studio** 的《第五人格》监管者首抓分析工具。程序会根据当前地图、4 名求生者的出生位置与角色、监管者出生位置、监管者角色以及监管者辅助特质，综合计算监管者开局更可能首追哪一名求生者，并输出首追概率、寻路距离、到达时间、首刀时间、击倒时间、模型系数、分析原因和行动建议。

程序采用图形化点击输入方式。启动后会打开 EasyX 窗口，用户通过鼠标选择地图、求生者位置、求生者角色、监管者位置、监管者角色和辅助特质，程序随后在右侧窗口显示完整分析结果，并将分析文本自动保存到 `result/` 文件夹。

本项目主要用于 C++ 面向对象程序设计、EasyX 图形界面练习、游戏数据建模、地图几何分析、最短路径计算、参数化模型设计和策略推荐展示。本项目不是《第五人格》官方工具，分析结果不代表官方结论，也不保证完全符合任何版本的真实对局环境。

## 当前版本说明

当前版本为 `v4.5`。本版本在 `v3.3` 的地图几何、可配置参数和评分模块基础上，重点加入了 **监管者辅助特质选择** 和 **追击时间模型**。最终概率不再只是由综合分直接决定，而是先估算监管者到达目标位置时间、首刀时间和击倒时间，再用击倒时间反向转换为首追概率。

当前版本主要特点如下：

1. 使用 EasyX 显示 `data/map` 中的 9 张地图图源。
2. 支持在地图原图上直接点击求生者和监管者位置。
3. 使用地图裁剪范围、白色像素识别、外墙外白底排除和墙体网格判断点击是否合法。
4. 使用墙体网格进行寻路距离计算，不再只依赖直线距离。
5. 使用求生者周边板窗资源修正追击收益。
6. 支持过山车、电车、荡绳、滑梯、船、地铁等特殊地图资源权重。
7. 支持队友互助修正，例如修机位靠近辅助位、救援位时降低首追收益。
8. 将角色原始 1-10 分折算为百分制，提高评分表达清晰度。
9. 使用 `score_calculator` 模块集中计算角色、监管者、地图、距离、克制、队友、辅助特质等分值。
10. 使用 `config/model_params.txt` 管理模型参数，修改参数后不需要重新编译程序。
11. 新增监管者辅助特质系统，支持聆听、失常、兴奋、巡视者、传送、窥视者、闪现、移形等特质。
12. 辅助特质会根据冷却时间、目标类型、板窗资源和距离结构修正首刀时间与击倒时间。
13. 右侧输出会显示到达时间、首刀时间、击倒时间以及辅助特质修正前后的时间差。
14. 使用 `print` 类生成新版结果文本，并自动保存到 `result/YYYY-MM-DD_HH-MM-SS.txt`。
15. 附带 `data/identityv_scores.xlsx`，用于整理角色、地图、辅助特质、模型参数和资料来源。
16. 附带 `tools/generate_excel.py`、`tools/generate_map_geometry.py` 和 `tools/image_recognition/generate_map_geometry.py`，用于重新生成数据表和地图几何数据。

## 与上一版本的主要不同

相比上一版本，本版本的核心变化是：**加入辅助特质，并把首追判断从“综合分模型”进一步升级为“预估追击时间模型”**。

| 对比项 | 上一版本 | 当前版本 `v4.5` |
|---|---|---|
| 首追核心依据 | 主要根据综合评分转换概率 | 先估算到达时间、首刀时间、击倒时间，再用击倒时间转换概率 |
| 辅助特质 | 未完整纳入首追流程 | 新增辅助特质选择，支持聆听、失常、兴奋、巡视者、传送、窥视者、闪现、移形 |
| 辅助特质影响 | 无或较弱 | 根据冷却时间、目标标签、距离、板窗资源修正首刀和击倒时间 |
| 评分模块 | 已有 `score_calculator` | 进一步加入辅助特质修正、击倒时间修正和文本说明 |
| 输出结果 | 输出概率、距离、角色系数、地图分、克制分等 | 新增到达时间、首刀时间、击倒时间、辅助技能修正秒数 |
| 结果生成 | 主要依赖旧输出模块 | 新增 `print` 类，专门负责结果文本排版与保存 |
| 模型参数 | 支持外部参数配置 | 新增追击时间相关参数，如移动速度、基础首刀时间、基础二刀时间、最短击倒时间 |
| 概率计算 | 以综合分做 softmax | 以“辅助特质修正后的击倒时间取反”做 softmax |
| 解释能力 | 能说明角色、地图、克制等原因 | 能进一步说明辅助特质为何生效或为何暂不生效 |
| 工程结构 | 数据、参数、评分、界面已初步分离 | 新增辅助特质类、辅助特质静态数据、输出 print 类，结构更完整 |

简单来说，`v3.3` 更侧重“地图几何 + 参数化评分”，`v4.5` 更侧重“**追击时间估算 + 辅助特质参与 + 更接近实战解释**”。

## 功能说明

本项目主要实现以下功能：

1. **地图选择**
   - 支持 9 张排位常见地图。
   - 程序左侧显示地图原图。
   - 程序内部根据地图几何数据判断点击位置、墙体、窗、板和距离。

2. **求生者信息输入**
   - 支持依次选择 4 名求生者的位置。
   - 支持依次选择 4 名求生者角色。
   - 地图上会标记 `S1`、`S2`、`S3`、`S4`。
   - 选中角色后，会在地图标记附近显示求生者名字。

3. **监管者信息输入**
   - 支持选择监管者出生点或当前所在位置。
   - 支持选择监管者角色。
   - 地图上会标记 `H` 表示监管者位置。

4. **辅助特质输入**
   - 支持选择监管者辅助特质。
   - 当前版本包含聆听、失常、兴奋、巡视者、传送、窥视者、闪现、移形。
   - 不同辅助特质会按照冷却时间和适配场景修正追击时间。

5. **返回与撤销**
   - 第 1 名求生者位置页可以返回地图选择。
   - 后续求生者位置页可以撤销上一名求生者。
   - 角色选择页可以撤销当前位置选择。
   - 监管者和辅助特质选择也支持返回上一层。
   - 避免选错后必须重启程序。

6. **首追概率计算**
   - 根据求生者角色价值、监管者追击风格、双方寻路距离、地图几何压力、特殊地图资源、队友互助、角色克制关系和辅助特质进行综合判断。
   - 先估算到达时间、首刀时间、击倒时间。
   - 再使用 softmax 将 4 名求生者的击倒时间转换为首追概率。
   - 输出监管者对每名求生者的首追意愿。

7. **结果展示**
   - 右侧区域显示完整分析文本。
   - 输出每名求生者的能力指标、出生地、首追概率、寻路距离、到达时间、首刀时间、击倒时间、计算原因和行动建议。
   - 输出监管者能力分析和辅助特质作用说明。

8. **结果保存**
   - 每次运行结束后自动生成分析结果文本。
   - 结果保存到 `result/` 文件夹。
   - 文件名按当前时间生成，不会覆盖上一次结果。

## 支持辅助特质

当前版本支持 8 个监管者辅助特质：

| 编号 | 辅助特质 | 模型作用 |
|---:|---|---|
| 1 | 聆听 | 偏信息确认，主要辅助判断方向，通常不直接压缩击倒时间 |
| 2 | 失常 | 偏控密码机，本模型中不直接修正首追击打时间 |
| 3 | 兴奋 | 克制控制位，对前锋、击球手、勘探员等控制型角色收益更高 |
| 4 | 巡视者 | 锁定目标，降低求生者转点与博弈空间 |
| 5 | 传送 | 解决远距离追击问题，对远点目标收益更高 |
| 6 | 窥视者 | 提供暴露与减速，配合控场型监管者收益更高 |
| 7 | 闪现 | 压缩板窗博弈时间，对强板窗区和高牵制角色收益更高 |
| 8 | 移形 | 支持位置交换和远点切入，适合改变追击路线 |

辅助特质不会无条件生效。程序会判断开局冷却时间，如果首刀或击倒发生在冷却结束之前，则不会假设辅助特质已经可用。

## 使用方法

程序所有输入均通过鼠标点击完成，不需要在终端输入。

输入顺序如下：

1. 在右侧点击本局地图。
2. 在左侧地图原图上点击第 1 名求生者所在位置。
3. 在右侧角色列表中点击第 1 名求生者角色。
4. 重复完成 4 名求生者的位置和角色选择。
5. 在左侧地图原图上点击监管者出生点或当前位置。
6. 在右侧角色列表中点击监管者角色。
7. 在右侧辅助特质列表中点击本局监管者辅助特质。
8. 查看右侧完整分析结果。
9. 在 `result/` 文件夹中查看自动保存的结果文本。

撤销规则如下：

- 第 1 名求生者位置页点击“返回选地图”，可以重新选择地图。
- 后续求生者位置页点击“撤销上一名”，会删除上一名已确认的求生者。
- 角色选择页点击“撤销位置”，会回到当前角色的位置选择。
- 监管者选择页可以返回上一层。
- 辅助特质选择页可以返回重新选择监管者。
- 只有有效地图范围内、外墙内部的白色空地可以点选；点到墙体、建筑、窗、板、门图标、文字、图例、外墙外白底或图片空白边时，程序会要求重新点击。

结果文件命名格式示例：

```text
result/2026-06-16_18-13-10.txt
```

每次输出都会新建一个按当前时间命名的文件，不会覆盖历史结果。

## 类与对象关系说明

本项目整体采用面向对象结构设计，核心思路是将“数据对象”“地图对象”“参数配置”“评分模型”“图形界面”“输出模块”和“程序控制器”分开。

### 1. 程序入口与控制器

```text
main()
└── first_chase_app
    ├── easyx_ui
    ├── data_repository
    ├── first_chase_analyzer
    │   └── score_calculator
    └── print
```

`main.cpp` 中的 `main()` 函数只负责设置控制台编码并创建 `first_chase_app` 对象运行程序。具体业务流程交给 `first_chase_app` 处理。

`first_chase_app` 是程序控制器，负责串联完整流程：

1. 初始化 EasyX 界面。
2. 读取地图选择。
3. 读取 4 名求生者的位置和角色。
4. 读取监管者位置和角色。
5. 读取监管者辅助特质。
6. 调用首追分析器计算结果。
7. 调用 `print` 生成结果文本并保存。
8. 调用界面类展示最终结果。

---

### 2. 求生者数据结构与求生者类

```text
survivor_static_data
└── survivor
```

`survivor_static_data` 是求生者静态数据结构，用于保存求生者的基础评分和标签。

主要字段包括：

| 字段名 | 含义 |
|---|---|

## 采用的算法说明

本项目不是简单地用固定规则判断首追目标，而是综合使用了多种算法和计算方法。

### 1. 百分制折算算法

求生者和监管者的能力原始数据使用 1-10 分。为了让不同角色之间的差异更明显，程序会把 1-10 分折算为百分制区间。

默认折算规则：

```text
1 分  -> 60 分
10 分 -> 100 分
```

折算公式为：

```text
percent_score = percent_min + (raw_score - 1) × (percent_max - percent_min) / 9
```

其中：

```text
percent_min = 60
percent_max = 100
percent_middle = 80
```

`percent_middle` 作为模型判断“中等水平”的基准线。

---

### 2. 角色指标评分算法

角色指标分主要考虑：

- 破译能力
- 牵制能力
- 辅助能力
- 救援能力
- 首抓价值
- 弱牵制标签

大致逻辑是：破译越高、牵制越弱、首抓价值越高，越容易被监管者优先追击。

示意公式：

```text
角色指标分 =
    破译压力
  + 首抓价值压力
  + 弱牵制压力
  + 辅助位压力
  + 救援位压力
  + 特殊标签修正
```

救援位如果本身牵制强，开局硬追通常不划算，因此会额外扣分；弱牵制角色会额外加分。

---

### 3. 监管者风格匹配算法

监管者风格分用于判断监管者的能力类型是否适合追某个求生者。

主要考虑：

- 监管者追击能力和求生者牵制能力的差值
- 监管者综合强度修正
- 监管者控场 / 信息能力对破译位的压制
- 监管者守椅能力对救援位的影响
- 监管者控场能力对辅助位的影响
- 慢追击监管者追强牵制角色的风险
- 快追击监管者追弱牵制角色的收益

例如，追击能力强的监管者更愿意抓弱牵制角色；控场和信息能力强的监管者更重视高破译角色。

---

### 4. 地图区域压力算法

地图区域压力主要来自 `map_data.h` 中的区域评分。

每个区域都有：

- `kite_score`：牵制舒适度
- `openness`：开阔程度
- `decode_value`：电机 / 转点价值

基础逻辑是：

```text
区域牵制越弱，首追分越高；
区域越开阔，首追分越高；
区域电机价值越高，压制收益越高。
```

示意公式：

```text
region_pressure =
    (10 - kite_score) × 1.8
  + (openness - 5) × 1.0
  + (decode_value - 5) × 0.9
```

---

### 5. 附近板窗资源修正算法

当前版本不只看区域，还会看点击点附近是否有窗和板。

程序从 `map_geometry_data.h` 中读取 `windows` 和 `pallets`，将窗和板看作地图线段，然后计算点击点到这些线段的距离。

主要逻辑是：

- 离窗越近，说明求生者附近有可用牵制资源，追击难度提高。
- 离板越近，说明求生者附近有可用板区资源，追击难度提高。
- 板窗资源会增加拖延时间，降低监管者直接首追收益。
- 资源分有上限，避免一个点附近资源过多导致评分失真。

---

### 6. 特殊地图资源评分算法

当前版本加入特殊地图资源对首追判断的影响。

主要资源包括：

| 地图 | 特殊资源 |
|---|---|
| 月亮河公园 | 过山车、滑梯、桥区 |
| 永眠镇 | 电车站 |
| 湖景村 | 大船 / 小船区域 |
| 不归林 | 树屋、荡绳、河道等转点资源 |

这些资源通常帮助求生者快速转点，因此默认会降低监管者首追意愿。如果监管者具有强机动、传送或反位移能力，则会抵消部分特殊地形收益。

---

### 7. 队友互助评分算法

程序会计算四名求生者之间的寻路距离，而不是简单直线距离。

主要逻辑如下：

- 修机位附近有辅助位或救援位，说明首追可能被补状态、挡刀或干扰，首追收益下降。
- 修机位如果孤立无援，监管者首追收益上升。
- 辅助位或救援位贴近修机位时，监管者可能先压辅助，拆掉修机与辅助联动结构。

该算法主要用于模拟职业赛或高配合对局中常见的“互保站位”和“拆联动”思路。

---

### 8. 角色克制标签算法

程序为监管者和求生者设置了标签，例如：

```text
anti_mobility
anti_item
anti_heal
anti_decoder
area_control
long_range
teleport
anti_loop
decoder
mobility
item_dependent
healer
stunner
kiter
weak_kiter
portal
stealth
rescue
assist
```

当监管者标签和求生者标签形成克制关系时，会增加追击分数。

例如：

- `anti_mobility` 克制 `mobility`
- `anti_item` 克制 `item_dependent`
- `anti_heal` 克制 `healer`
- `anti_decoder` 克制 `decoder`
- `anti_loop` 克制 `kiter`
- `teleport` 对 `stealth` 有一定压制
- `camp` 对 `rescue` 有一定压制

此外，部分特殊监管者和求生者类型还设置了额外修正，例如摄影师、疯眼、隐士对破译位的压制，或强牵制角色面对慢追击监管者时降低首追收益。

---

### 9. 辅助特质修正算法

当前版本新增辅助特质模型。每个辅助特质包含：

- 开局冷却时间
- 对首刀时间的基础修正
- 对击倒时间的基础修正
- 辅助特质标签
- 文字效果说明

辅助特质不会在冷却结束前提前生效。程序会分别判断：

```text
如果正常首刀时间 < 辅助特质冷却时间，则辅助特质不修正首刀时间；
如果正常击倒时间 < 辅助特质冷却时间，则辅助特质不修正击倒时间。
```

不同辅助特质还有额外适配逻辑：

- 闪现适合处理强板窗和高牵制点，板窗资源越多，压缩时间越明显。
- 传送和移形适合处理远距离目标，距离越远，抵消赶路时间越多。
- 兴奋主要克制控制位，对前锋、击球手、勘探员等角色收益更高。
- 巡视者对缺少位移的角色更稳定，对高机动角色收益会打折。
- 窥视者配合控场型监管者收益更高。
- 失常偏控密码机，不直接缩短开局追击击倒时间。
- 聆听偏信息确认，更多影响方向判断，不直接压缩击倒时间。

---

### 10. 追击时间估计算法

当前版本最终概率以“预估击倒时间”为主要依据。

程序先计算监管者从当前位置到求生者位置的寻路距离，再估算：

```text
到达时间 = 寻路距离 / 监管者移动速度
```

之后根据角色指标、监管者风格、克制关系、地图难度、队友互保和板窗资源，修正首刀时间与击倒时间。

简化理解：

```text
追击优势越大，首刀和击倒时间越短；
地图越难追、板窗越多、队友互保越强，首刀和击倒时间越长；
辅助特质如果适配当前目标，会进一步压缩首刀或击倒时间。
```

最终模型会得到：

```text
arrival_time_seconds
first_hit_time_seconds
first_hit_time_with_trait_seconds
down_time_seconds
down_time_with_trait_seconds
```

概率计算时，使用辅助特质修正后的击倒时间作为核心依据：

```text
raw_score = - down_time_with_trait_seconds
```

也就是说，预估击倒时间越短，`raw_score` 越高，首追概率越高。

---

### 11. 地图图像识别与几何数据生成算法

当前版本使用 `tools/image_recognition/generate_map_geometry.py` 从地图图源中提取几何信息，同时保留 `tools/generate_map_geometry.py` 作为旧入口。

主要步骤包括：

1. 对地图图片设置实际对局区域裁剪框。
2. 将裁剪后的地图划分为 90 × 90 网格。
3. 根据像素颜色判断墙体区域。
4. 根据蓝色像素识别窗。
5. 根据绿色像素识别板。
6. 使用连通块算法合并同一段窗或板。
7. 将窗、板和墙体网格转换为 0-1000 相对地图坐标。
8. 生成 `include/map_geometry_data.h`，供 C++ 程序使用。

其中，窗和板识别使用颜色阈值和连通块统计；墙体识别使用灰度、颜色差和网格内墙体像素占比判断。

---

### 12. 点击合法性判定算法

为了避免用户点到墙体、文字、图例、外墙外空白或不可走区域，本项目采用多层点击判定：

1. 判断鼠标是否在地图画布范围内。
2. 将屏幕坐标转换为 0-1000 地图相对坐标。
3. 判断是否在地图有效裁剪范围内。
4. 判断原图对应像素是否接近白色可走区域。
5. 判断是否属于外墙外侧的大块白底。
6. 判断该点对应的几何网格是否为墙体或障碍。
7. 只有通过全部判定，才允许作为求生者或监管者位置。

这样可以尽量避免点到不可走区域，提升分析输入的合理性。

---

### 13. 最短路径 / 寻路距离算法

当前版本的距离不使用区域中心点直线距离，而是使用地图网格进行寻路。

主要步骤：

1. 将监管者位置和求生者位置转换为 90 × 90 网格下的起点和终点。
2. 如果起点或终点落在墙体网格上，就用 BFS 思想寻找最近的可走网格。
3. 使用带优先队列的 Dijkstra 最短路径算法计算两点之间的路径距离。
4. 支持 8 个方向移动，包括上下左右和斜向移动。
5. 斜向移动时检查两侧相邻格，避免从墙角斜穿过去。
6. 每一步距离根据地图估算真实宽高换算为米制距离。
7. 如果寻路失败，则退回到欧氏距离并乘以修正系数。

距离越远，监管者开局首追该求生者的意愿越低。

---

### 14. Softmax 概率归一化算法

4 名求生者都会得到一个原始分 `raw_score`。当前版本中：

```text
raw_score = - down_time_with_trait_seconds
```

也就是击倒时间越短，原始分越高。

为了将原始分转换成首追概率，程序使用 softmax 算法：

```text
probability_i = exp((score_i - best_score) / softmax_divisor)
              / sum(exp((score_j - best_score) / softmax_divisor))
```

其中减去 `best_score` 是为了避免指数计算过大；`softmax_divisor` 用于控制概率差距，不让小分差被过度放大。

最终输出为百分比，表示监管者更可能首追哪名求生者。

---

### 15. 参数文件解析算法

`model_params_loader` 会读取 `config/model_params.txt`，按行解析参数。

解析规则如下：

1. 去掉 `#` 后面的注释。
2. 判断当前行是否包含 `=`。
3. 提取 `key=value`。
4. 去掉首尾空白字符。
5. 将数值转成 `double`。
6. 根据参数名写入 `model_params`。
7. 未知参数会被忽略，便于后续扩展。

这个设计使得用户修改模型权重时不需要重新编译代码。

---

### 16. 文本自动换行与滚动显示算法

右侧结果区内容较多，程序会对中文文本进行自动换行处理。

实现思路：

1. 按 GBK 中文字符宽度拆分文本。
2. 使用 EasyX 的文字宽度判断当前行能否继续加入字符。
3. 超出宽度时自动换行。
4. 结果页支持滚动查看，避免长文本显示不完整。
5. 输出说明部分还会按标点切分，尽量让每行控制在约 50 个字符以内，避免中文和数字被挤出显示区域。

## 设计特点

### 1. 主函数较简洁

`main.cpp` 只负责设置编码并启动 `first_chase_app`，避免所有逻辑堆在主函数中。

### 2. 输入输出和算法分离

`easyx_ui` 只负责界面，`first_chase_analyzer` 和 `score_calculator` 负责算法，`print` 负责结果文本，便于后续修改和维护。

### 3. 静态数据集中管理

求生者、监管者、辅助特质、地图区域和地图几何数据分别放在独立头文件中，便于后续更新角色、辅助特质、地图和评分。

### 4. 模型参数外置

模型权重集中放在 `config/model_params.txt` 中，修改参数后无需重新编译程序。

### 5. 支持图形化点击输入

用户不需要在控制台输入编号，直接通过 EasyX 界面点击位置、角色和辅助特质即可完成输入。

### 6. 支持地图几何路径计算

当前版本能够根据墙体网格计算路径距离，而不是只根据直线距离判断远近。

### 7. 支持追击时间估算

当前版本会估算到达时间、首刀时间和击倒时间，使输出更接近“监管者为什么会首追这个人”的实战解释。

### 8. 输出结果可保存

每次分析结果都会保存为独立文本文件，便于复盘和对比。

## 当前不足与未完成内容

当前版本核心功能已经完成，但仍有一些没有完成或可以继续完善的地方：

1. **没有自动读取 Excel 数据**
   - `data/identityv_scores.xlsx` 目前主要作为整理表。
   - 程序实际使用的数据仍写在 `include/survivor_data.h`、`include/hunter_data.h`、`include/auxiliary_trait_data.h`、`include/map_data.h` 和 `include/map_geometry_data.h` 中。
   - 后续可以增加从 Excel、CSV 或 JSON 自动读取数据的功能。

2. **权重已有配置文件，但仍没有图形化设置界面**
   - 当前可以通过 `config/model_params.txt` 修改权重。
   - 后续可以增加“权重设置界面”，允许用户在程序中直接调整模型参数。

3. **地图几何识别仍有主观修正**
   - 当前地图几何数据来自图片颜色识别和手动裁剪框。
   - 对部分地图的特殊道路、白底、图标和复杂建筑区域仍可能需要人工修正。

4. **暂未支持对局录像或截图自动识别**
   - 当前需要用户手动选择角色和位置。
   - 后续可以扩展为读取截图、录像或 OCR 识别，但当前版本未实现。

5. **辅助特质仍是简化模型**
   - 当前辅助特质主要修正首刀和击倒时间。
   - 后续可以继续加入天赋、人格、具体监管者技能前摇、后摇、存在感阶段等因素。

6. **结果输出格式仍可扩展**
   - 当前主要输出 `.txt` 文本文件。
   - 后续可以增加 Markdown、HTML、CSV 或图片报告导出。

7. **跨平台能力有限**
   - 当前依赖 Windows 和 EasyX。
   - Linux、macOS 暂不支持直接运行。

8. **数据版本需要持续维护**
   - 《第五人格》角色、地图、辅助特质和版本环境会更新。
   - 本项目评分数据需要根据实际版本继续调整。

9. **模型仍属于主观策略模型**
   - 当前模型根据角色机制、地图理解和手工权重进行估计。
   - 后续可以结合真实对局数据进行统计校准。
